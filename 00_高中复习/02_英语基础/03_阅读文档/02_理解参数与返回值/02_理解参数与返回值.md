## 正文讲解

### 1. 从一个熟悉的场景说起

想象你拿到了一台新微波炉的说明书。说明书上写着：

> **加热食物**：将食物放入（必须），设置时间（必须），选择功率（可选，默认为中火）。完成后，微波炉发出提示音（返回结果）。

这就是日常生活中的"函数文档"——它告诉你需要提供什么（参数），哪些是必须的、哪些是可选的，以及操作完成后会得到什么（返回值）。编程文档中描述函数的方式与此完全一样，只是用了更规范的格式。

### 2. 函数签名：文档中的"一句话说明"

打开任何一份 Python 文档，在 API 参考部分，每个函数的描述都以一行 **函数签名（Function Signature）** 开头。函数签名就像函数的"名片"，用最精炼的方式告诉你这个函数叫什么名字、需要哪些参数。

我们来看 Python 内置函数 `print` 的签名：

```
print(*objects, sep=' ', end='\n', file=sys.stdout, flush=False)
```

乍一看信息量不小，但其实它遵循一个固定格式：

```
函数名(参数1, 参数2=默认值, 参数3=默认值, ...)
```

让我们拆解 `print` 的签名：

| 组成部分 | 内容 | 含义 |
| -------- | ---- | ---- |
| 函数名 | `print` | 这个函数叫 print |
| `*objects` | 星号开头的参数 | 可以传入任意数量的要打印的对象 |
| `sep=' '` | 带默认值的参数 | 多个对象之间的分隔符，默认是一个空格 |
| `end='\n'` | 带默认值的参数 | 打印结束后添加的字符，默认是换行 |
| `file=sys.stdout` | 带默认值的参数 | 输出目标，默认是屏幕 |
| `flush=False` | 带默认值的参数 | 是否立即刷新输出，默认否 |

### 3. 必选参数与可选参数

在文档中区分 **必选参数（Required Parameter）** 和 **可选参数（Optional Parameter）** 是最关键的阅读技能之一。规则很简单：

- **有默认值的参数是可选的**——调用时你可以不传这个参数，函数会使用默认值
- **没有默认值的参数是必选的**——调用时你必须提供这个参数，否则会报错

让我们看另一个例子——Python 内置函数 `range`：

```
range(stop)
range(start, stop[, step])
```

这里出现了一种新的表示法：**方括号 `[, step]`** 表示 `step` 是可选参数。这是文档中非常常见的约定。

| 标记方式 | 含义 | 示例 |
| -------- | ---- | ---- |
| `参数名` | 必选参数 | `range(stop)` 中的 `stop` |
| `参数名=默认值` | 可选参数（有默认值） | `print(end='\n')` 中的 `end` |
| `[参数名]` 或 `[, 参数名]` | 可选参数（方括号标记） | `range(start, stop[, step])` 中的 `step` |
| `*args` | 接受任意数量的位置参数 | `print(*objects)` |
| `**kwargs` | 接受任意数量的关键字参数 | 常见于高级函数中 |

### 4. 常见类型标注

现代 Python 文档越来越多地使用 **类型标注（Type Annotation）** 来说明参数应该是什么类型的数据。虽然你还没有正式学习编程，但提前认识这些类型名称会让你阅读文档时更加顺畅。

以下是最常见的类型标注：

| 类型标注 | 含义 | 日常类比 |
| -------- | ---- | -------- |
| `str` | 字符串（文本） | 一段文字，如 `"hello"` |
| `int` | 整数 | 整数，如 `42` |
| `float` | 浮点数（小数） | 小数，如 `3.14` |
| `bool` | 布尔值（真/假） | 开关，只有 `True` 或 `False` |
| `list` | 列表 | 一组有序的东西，如购物清单 |
| `dict` | 字典 | 一组键值对，如通讯录（名字→电话） |
| `None` | 空值 | 表示"什么都没有" |
| `Optional[X]` | 可选类型 | 可以是类型 $X$ ，也可以是 `None` |
| `Union[X, Y]` | 联合类型 | 可以是类型 $X$ ，也可以是类型 $Y$ |

例如在文档中你可能会看到这样的描述：

```
参数 filepath: str or Path
```

这表示 `filepath` 这个参数可以是一个字符串 `str`，也可以是一个 `Path` 对象。

### 5. 阅读 "Parameters" 和 "Returns" 部分

大多数 Python 库的文档在描述函数时，会有两个标准化的小节：**Parameters**（参数）和 **Returns**（返回值）。这两个部分是你最需要关注的核心内容。

让我们看一个来自 NumPy 文档的真实例子—— `numpy.zeros` 函数：

```
numpy.zeros(shape, dtype=float, order='C')

Parameters
----------
shape : int or tuple of ints
    Shape of the new array, e.g., (2, 3) or 2.
dtype : data-type, optional
    The desired data-type for the array. Default is numpy.float64.
order : {'C', 'F'}, optional
    Whether to store multi-dimensional data in row-major (C-style)
    or column-major (Fortran-style) order in memory.

Returns
-------
out : ndarray
    Array of zeros with the given shape, dtype, and order.
```

我们来逐条解读这个格式：

**Parameters 部分**的每一条都遵循相同格式：

```
参数名 : 类型, 是否可选
    描述文字。默认值说明。
```

- `shape : int or tuple of ints` → 参数名是 `shape`，类型是整数或整数元组，**没有标注 optional，所以是必选的**
- `dtype : data-type, optional` → 参数名是 `dtype`，明确标注了 `optional`（可选），默认值是 `numpy.float64`
- `order : {'C', 'F'}, optional` → 花括号 `{'C', 'F'}` 表示只能从这几个值中选一个

**Returns 部分**告诉你函数执行后会给你什么：

- `out : ndarray` → 返回一个 `ndarray` 类型的数组
- 描述文字进一步说明：这个数组充满零值，具有你指定的形状、类型和排列方式

### 6. 另一个真实例子：pandas.read_csv

让我们再看一个稍复杂的例子。 `pandas.read_csv` 是数据处理中最常用的函数之一，它的文档签名非常长：

```
pandas.read_csv(filepath_or_buffer, *, sep=',', header='infer',
                names=None, index_col=None, usecols=None, dtype=None,
                engine=None, skiprows=None, nrows=None, ...)
```

面对这么长的签名，不要慌张。阅读策略是：

1. **找到必选参数**：`filepath_or_buffer` 是唯一没有默认值的参数，所以它是必选的——你必须告诉函数要读哪个文件
2. **找到你关心的可选参数**：根据你的需求，浏览参数名称——`sep`（分隔符）、`header`（标题行）、`nrows`（读取行数）等，找到你需要修改的那个
3. **忽略不相关的参数**：那个 `...` 省略号后面还有几十个参数！但你通常只需要用到其中两三个。不需要理解每一个参数

签名中的 `*` 星号是一个特殊标记——它表示在 `*` 之后的所有参数都必须用 `参数名=值` 的方式传入（叫作 **关键字参数（Keyword Argument）**），不能只传值。

### 7. 参数相关的常见英文术语

最后，我们汇总文档中与参数和返回值相关的常见英文术语：

| 英文术语 | 中文含义 | 说明 |
| -------- | -------- | ---- |
| Parameter | 参数 | 函数定义中的变量 |
| Argument | 参数/实参 | 调用函数时传入的具体值 |
| Required | 必选的 | 必须提供 |
| Optional | 可选的 | 可以不提供，使用默认值 |
| Default | 默认值 | 不传参数时使用的值 |
| Returns | 返回值 | 函数执行后给你的结果 |
| Raises | 抛出异常 | 出错时产生的错误 |
| Type | 类型 | 数据的类别 |
| Keyword argument | 关键字参数 | 用 `名称=值` 方式传入 |
| Positional argument | 位置参数 | 按位置顺序传入 |
| Deprecated | 已废弃 | 参数或功能不再推荐使用 |
| See Also | 另请参阅 | 相关函数推荐 |

## 动手实践

现在让我们练习阅读真实的函数文档。请打开 Python 官方文档中 `open` 函数的页面（https://docs.python.org/3/library/functions.html#open），找到它的函数签名：

```
open(file, mode='r', buffering=-1, encoding=None, errors=None,
     newline=None, closefd=True, opener=None)
```

尝试回答以下问题：

1. 哪个参数是必选的？
2. `mode` 参数的默认值是什么？这意味着什么？
3. `encoding` 参数的默认值是 `None`，它是必选的还是可选的？
4. 一共有几个可选参数？

**预期答案**：
```
1. file 是必选的（没有默认值）
2. mode 的默认值是 'r'，意味着默认以只读模式打开文件
3. encoding 是可选的（有默认值 None）
4. 7 个可选参数（mode, buffering, encoding, errors, newline, closefd, opener）
```

> 💡 **提示**：你不需要理解每个参数的具体作用。现在的重点是练习从签名中提取"哪些必选、哪些可选、默认值是什么"这些关键信息。

## 参数描述常用语块

在函数文档的参数和返回值描述中，以下语块极为常见：

| 语块 | 中文含义 | 示例 |
| ---- | -------- | ---- |
| `optional` / `default: X` | 可选参数/默认值为 X | `axis (int, optional, default: 0)` |
| `must be` / `should be` | 必须是/应该是 | `must be a positive integer` |
| `one of` | …之一 | `one of {'linear', 'rbf', 'poly'}` |
| `if None` / `if not specified` | 如果未指定 | `if None, defaults to the identity matrix` |
| `Returns ... or None` | 返回…或 None | 函数可能不返回值 |
| `Raises ValueError if` | 如果…则抛出 ValueError | 描述错误触发条件 |
| `array-like` / `array_like` | 类数组对象 | 接受列表、元组、ndarray 等 |
| `shape (n, m)` | 形状为 (n, m) | 描述矩阵/数组的维度 |
| `of type X` / `X or Y` | 类型为 X / X 或 Y | 描述参数的数据类型 |
| `str, int, float, bool` | 字符串、整数、浮点数、布尔值 | 常见的类型标注 |

> 💡 **阅读技巧**：看函数文档时，先看参数类型（type）和默认值（default），再看文字描述。80% 的情况下，类型和默认值就足以理解一个参数的用法。

## 典型误区

| 误区 | 正确理解 |
| ---- | -------- |
| 每个参数都要理解才能使用函数 | 大多数函数有很多可选参数，你只需要关注必选参数和少数几个与你需求相关的可选参数 |
| `Optional` 和"可选参数"是同一个概念 | `Optional[str]` 是类型标注，表示"可以是 `str` 也可以是 `None`"；"可选参数"是指调用时可以不传的参数。两者不同 |
| 默认值是 `None` 就表示这个参数没用 | `None` 作为默认值通常表示"自动选择"或"不使用此功能"，函数内部会根据情况处理 |
| `*args` 和 `**kwargs` 很复杂，看到就跳过 | 它们只是表示"可以传任意多个参数"。初学阶段遇到它们可以先忽略，不影响基本使用 |

## 练习题

### 练习 1：解读函数签名（难度：⭐）

阅读以下函数签名，回答问题：

```
len(s)
```

1. 这个函数叫什么名字？
2. 有几个参数？
3. 这个参数是必选的还是可选的？

<details>
<summary>💡 提示</summary>

观察参数 `s` 后面有没有 `=默认值`。

</details>

<details>
<summary>✅ 参考答案</summary>

1. 函数名是 `len`
2. 只有 1 个参数：`s`
3. `s` 是**必选的**——它没有默认值，你必须告诉 `len` 要计算什么东西的长度

</details>

### 练习 2：识别参数类型（难度：⭐⭐）

阅读以下文档片段，回答问题：

```
Parameters
----------
data : array-like or dict
    The input data.
index : Index or array-like, optional
    Row labels for the DataFrame.
columns : Index or array-like, optional
    Column labels for the DataFrame.
dtype : str, numpy.dtype, or ExtensionDtype, optional
    Data type to force.
```

1. 哪个参数是必选的？
2. `index` 参数可以接受什么类型的数据？
3. 有几个可选参数？

<details>
<summary>💡 提示</summary>

标注了 `optional` 的是可选参数；没有标注的是必选参数。

</details>

<details>
<summary>✅ 参考答案</summary>

1. `data` 是必选的——它是唯一没有标注 `optional` 的参数
2. `index` 可以接受 `Index` 或 `array-like`（类数组）类型的数据
3. 3 个可选参数：`index`、`columns`、`dtype`

</details>

### 练习 3：从文档到调用（难度：⭐⭐）

阅读以下函数签名和参数说明：

```
sorted(iterable, /, *, key=None, reverse=False)

Parameters
----------
iterable : 可迭代对象
    要排序的数据。
key : function, optional
    用于提取比较键的函数。
reverse : bool, optional
    如果为 True，则按降序排列。默认为 False（升序）。

Returns
-------
list
    排序后的新列表。
```

回答：如果你想对一组数据进行**降序排列**，你需要修改哪个参数？把它设置为什么值？

<details>
<summary>💡 提示</summary>

寻找与"排列顺序"相关的参数，阅读它的描述。

</details>

<details>
<summary>✅ 参考答案</summary>

需要将 `reverse` 参数设置为 `True`。根据文档描述，`reverse=True` 表示按降序排列。由于 `reverse` 在 `*` 之后，所以调用时需要用关键字参数的方式：`sorted(数据, reverse=True)`。

</details>
