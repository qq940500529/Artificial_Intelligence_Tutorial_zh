## 概述

在 [并发编程](../07_并发编程/) 中，我们学会了如何让程序同时做多件事。但并发编程最常见的应用场景之一，就是**网络通信**——一边等待远程服务器响应，一边处理其他任务。

网络编程是连接"单机程序"与"互联网世界"的桥梁。无论是调用 ChatGPT 的 API、从网页抓取数据、还是构建一个数据标注服务，都离不开对网络协议和 HTTP 通信的理解。本主题将带你从最底层的 socket 编程开始，逐步上升到 HTTP 客户端、Web API 调用，最后学会用 Python 构建一个简单的 Web 服务。

掌握这些技能后，你将具备与外部系统交互的基本能力——这是数据采集、模型部署和 AI 工程化的重要前置技能。

## 学习建议

- **学习顺序**：建议按照 01 → 04 的顺序学习。01 socket 编程基础帮助你理解网络通信的底层原理；02 HTTP 客户端是日常使用最频繁的技能；03 Web API 调用在真实项目中无处不在；04 简单 Web 服务则让你从"调用者"变成"提供者"。
- **时间分配**：每个知识点预计 40–60 分钟（含动手实践），整个主题约需 3–4 小时。
- **学习方法**：网络编程必须动手实践。建议开两个终端窗口——一个运行服务端，另一个运行客户端——亲眼看到数据在两端之间流动。
- **常见困难点**：
  - socket 编程中，`bind`、`listen`、`accept`、`connect` 的调用顺序容易搞混——记住**服务端先 bind-listen-accept，客户端直接 connect**。
  - HTTP 响应的编码问题常导致中文乱码——注意检查响应头中的 `Content-Type` 和 `charset`。
  - API 认证方式多样（API Key、Bearer Token、OAuth），需要根据文档选择正确的方式。
