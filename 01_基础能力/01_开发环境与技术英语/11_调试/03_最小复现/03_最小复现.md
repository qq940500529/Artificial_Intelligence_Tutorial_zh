## 正文讲解

### 1. 什么是最小复现？

你向同事描述一个 bug："我的程序运行到某个地方就出错了。"同事回答："把你的代码发给我看看。"你发过去一个 5000 行的项目——同事看了三秒钟就关掉了。

**最小复现（Minimal Reproducible Example, MRE）** 是能够触发同一个 bug 的 **最短** 代码。它剥离了所有与问题无关的代码、配置和数据，只保留触发 bug 必需的最小集合。

```mermaid
graph LR
    A["5000 行项目"] --> B["剥离无关模块"]
    B --> C["简化数据"]
    C --> D["移除依赖"]
    D --> E["最小复现<br/>10-30 行"]
    
    style E fill:#4CAF50,color:#fff
```

> 📌 **图解说明**：从复杂项目到最小复现的简化过程。每一步都移除一些不影响 bug 触发的内容，最终得到最短的复现代码。

最小复现的价值不仅在于方便别人帮你排查——**构建最小复现的过程本身就是调试** 。超过一半的 bug 会在构建最小复现的过程中自然暴露。

### 2. 好的最小复现的标准

一个高质量的最小复现应该满足以下条件：

| 标准 | 说明 |
| ---- | ---- |
| **最小（Minimal）** | 不包含任何与问题无关的代码 |
| **完整（Complete）** | 复制粘贴就能运行，不需要额外的文件或配置 |
| **可复现（Reproducible）** | 每次运行都能触发同一个问题 |

```python
# ❌ 不好的 bug 报告
# "我的程序在处理数据时报错了，代码在附件里（2000行）"

# ✅ 好的 bug 报告
"""
Python 3.11 中，对包含 None 的列表排序时报 TypeError：

data = [3, None, 1, 2]
data.sort()  # TypeError: '<' not supported between instances of 'NoneType' and 'int'

期望行为：None 被放在列表末尾
实际行为：抛出 TypeError
"""
```

### 3. 构建最小复现的系统方法

#### 方法一：自底向上（从零构建）

从空文件开始，逐步添加代码，直到复现问题：

```python
# 步骤 1：最简单的调用
result = my_function(data)  # 能复现吗？

# 步骤 2：添加必要的数据
data = [{"name": "test", "value": None}]
result = my_function(data)  # 能复现吗？

# 步骤 3：添加必要的函数定义
def my_function(data):
    return [item['value'] * 2 for item in data]  # TypeError!

data = [{"name": "test", "value": None}]
result = my_function(data)
# 找到了！问题在于 value 为 None 时的乘法操作
```

#### 方法二：自顶向下（逐步剥离）

从完整程序开始，每次删除一部分代码，检查问题是否还能复现：

```python
# 原始代码：300 行
# 步骤 1：删除 UI 层 → 还能复现？是 → 继续
# 步骤 2：删除数据库层，用硬编码数据 → 还能复现？是 → 继续
# 步骤 3：删除日志和配置 → 还能复现？是 → 继续
# 步骤 4：简化函数逻辑 → 还能复现？否 → 上一步删的代码有关
# 步骤 5：把步骤 3 删的代码加回一半 → 还能复现？是 → 继续
# ...最终定位到问题代码
```

#### 方法三：二分法

当不确定哪段代码导致问题时，用二分法快速缩小范围：

```python
# 100 行代码，注释掉后半段
# 前 50 行能复现吗？
#   是 → 注释掉前半的后半（26-50行），看前25行能不能复现
#   否 → 问题在后 50 行，注释掉前半段，只看后50行

# 每次排除一半，100行代码最多需要 7 次就能定位到具体行
# （log2(100) ≈ 7）
```

### 4. 实战：构建最小复现

假设你遇到这样一个 bug——程序在处理某些数据时结果不正确：

```python
# 原始代码（简化版，实际可能有几百行）
import json

def load_config(filepath):
    with open(filepath) as f:
        return json.load(f)

def preprocess(data, config):
    threshold = config.get('threshold', 0.5)
    processed = []
    for item in data:
        if item['score'] > threshold:
            processed.append({
                'name': item['name'],
                'score': round(item['score'], 2),
                'label': 'high' if item['score'] > 0.8 else 'medium'
            })
    return processed

def calculate_stats(processed):
    if not processed:
        return {'count': 0, 'avg': 0}
    total = sum(p['score'] for p in processed)
    return {
        'count': len(processed),
        'avg': total / len(processed)
    }

# Bug: 某些情况下 avg 精度不对
```

构建最小复现的过程：

```python
# 步骤 1：去掉文件读取，硬编码数据
data = [
    {'name': 'A', 'score': 0.7},
    {'name': 'B', 'score': 0.9},
    {'name': 'C', 'score': 0.1},
]

# 步骤 2：去掉预处理，直接测试统计函数
processed = [{'score': 0.7}, {'score': 0.9}]
total = sum(p['score'] for p in processed)
avg = total / len(processed)
print(f"total={total}, avg={avg}")
# total=1.5999999999999998, avg=0.7999999999999999

# 步骤 3：进一步简化
print(0.7 + 0.9)  # 1.5999999999999998
# 找到了！这是浮点精度问题，不是代码 bug
```

### 6. 常用简化技巧

| 技巧 | 说明 |
| ---- | ---- |
| 硬编码数据 | 用字面量替换文件读取、数据库查询、API 调用 |
| 移除导入 | 每个 import 都是潜在的依赖，尝试删掉不相关的 |
| 内联函数 | 把被调用的函数展开，减少层次 |
| 简化数据 | 用最少的数据触发问题（一条记录 vs 一万条） |
| 去掉异常处理 | 暂时移除 try/except，让错误自然暴露 |
| 替换外部依赖 | 用标准库或内置类型替代第三方库的对象 |

## 动手实践

```python
# 文件：code/minimal_repro_demo.py
# 最小复现方法练习

# 场景：一个数据处理管道的结果不符合预期
# 任务：通过逐步简化找到根本原因

def load_data():
    """模拟从数据库加载数据"""
    return [
        {'id': 1, 'values': [10, 20, 30]},
        {'id': 2, 'values': [5, 15, 25]},
        {'id': 3, 'values': []},  # 空列表！
        {'id': 4, 'values': [40, 50]},
    ]


def compute_averages(records):
    """计算每条记录的平均值"""
    results = []
    for record in records:
        avg = sum(record['values']) / len(record['values'])
        results.append({'id': record['id'], 'avg': avg})
    return results


def filter_above_threshold(records, threshold=15):
    """筛选平均值大于阈值的记录"""
    return [r for r in records if r['avg'] > threshold]


# 完整管道
try:
    data = load_data()
    averages = compute_averages(data)
    filtered = filter_above_threshold(averages)
    print("结果:", filtered)
except ZeroDivisionError:
    print("发现 bug！某条记录的 values 为空列表")
    
    # 构建最小复现
    print("\n=== 最小复现 ===")
    print("sum([]) / len([]) 会导致 ZeroDivisionError")
    try:
        sum([]) / len([])
    except ZeroDivisionError as e:
        print(f"错误: {e}")
    
    # 修复
    print("\n=== 修复方案 ===")
    def safe_average(values):
        return sum(values) / len(values) if values else 0
    
    print(f"safe_average([10, 20, 30]) = {safe_average([10, 20, 30])}")
    print(f"safe_average([]) = {safe_average([])}")
```

**运行说明**：
- 环境要求：Python 3.10+
- 运行命令：`python code/minimal_repro_demo.py`

**预期输出**：
```
发现 bug！某条记录的 values 为空列表

=== 最小复现 ===
sum([]) / len([]) 会导致 ZeroDivisionError
错误: division by zero

=== 修复方案 ===
safe_average([10, 20, 30]) = 20.0
safe_average([]) = 0
```

## 典型误区

| 误区 | 正确理解 |
| ---- | -------- |
| 最小复现只是为了给别人看的 | 构建最小复现的过程本身就是最强大的调试技术——大多数 bug 会在简化过程中自然暴露 |
| 直接把整个项目发给别人求助 | 没有人愿意阅读几千行无关代码。最小复现能让你更快得到帮助 |
| 如果能在本地复现就不需要最小复现 | 最小复现帮助你理解 bug 的 **根本原因** ，而不只是 **触发条件** |
| 构建最小复现太浪费时间 | 对于复杂 bug，构建最小复现通常比盯着代码猜测要快得多 |

## 练习题

### 练习 1：从错误日志构建最小复现（难度：⭐）

给定以下错误信息，编写最小复现代码：

```
TypeError: unsupported operand type(s) for +: 'int' and 'str'
  File "app.py", line 42, in process
    total += item['amount']
```

<details>
<summary>💡 提示</summary>

错误说明 `item['amount']` 有时是字符串而不是数字。构建一个包含混合类型的最小数据集来复现。

</details>

<details>
<summary>✅ 参考答案</summary>

```python
# 最小复现
total = 0
items = [{'amount': 100}, {'amount': '200'}]  # 第二个是字符串
for item in items:
    total += item['amount']  # TypeError!

# 修复
total = 0
for item in items:
    total += int(item['amount'])  # 强制转换
print(total)  # 300
```

</details>

### 练习 2：二分法定位（难度：⭐⭐）

下面的函数在某些输入上结果不正确。用二分法注释代码来定位 bug：

```python
def transform(data):
    # 步骤1: 去重
    data = list(set(data))
    # 步骤2: 排序
    data.sort()
    # 步骤3: 过滤负数
    data = [x for x in data if x >= 0]
    # 步骤4: 归一化到 0-1
    max_val = max(data)
    data = [x / max_val for x in data]
    return data

# 测试
print(transform([3, 1, -2, 4, 1, 3]))  # 期望：[0.25, 0.75, 1.0]
print(transform([0, 0, 0]))  # 这里会出错吗？
```

<details>
<summary>💡 提示</summary>

当所有值都是 0 时，步骤 1 去重后只有一个 0，步骤 4 的 `max_val` 也是 0，导致除以零。

</details>

<details>
<summary>✅ 参考答案</summary>

```python
# 最小复现
data = [0]
max_val = max(data)  # 0
result = [x / max_val for x in data]  # ZeroDivisionError!

# 修复
def transform(data):
    data = list(set(data))
    data.sort()
    data = [x for x in data if x >= 0]
    max_val = max(data) if data else 1
    if max_val == 0:
        return [0.0] * len(data)
    return [x / max_val for x in data]

print(transform([0, 0, 0]))  # [0.0]
```

</details>
