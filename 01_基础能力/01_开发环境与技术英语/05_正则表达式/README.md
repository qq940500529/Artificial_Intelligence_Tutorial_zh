## 概述

在数据处理、文本清洗和日志分析等任务中，我们经常需要从大量文本中查找、提取或替换符合特定规则的内容。比如：从一段日志中提取所有 IP 地址，从网页源码中抓取所有链接，或者将不规范的日期格式统一转换成标准格式。如果只用 [字符串方法](../02_字符串与编码/01_字符串方法与格式化/01_字符串方法与格式化.md) 中学过的 `find()`、`replace()`、`split()` 等基础操作，代码会变得冗长且难以维护。**正则表达式（Regular Expression, Regex）** 正是为解决这类问题而设计的强大工具——它用一套简洁的模式语法来描述文本的结构特征，让你可以用一行表达式完成原本需要数十行代码才能实现的文本匹配与处理任务。

Python 通过标准库的 `re` 模块提供了完整的正则表达式支持。本主题将从最基础的模式语法讲起，逐步深入到匹配搜索与替换、分组与断言等高级特性，最终通过实战案例帮助你将正则表达式灵活运用到日志解析、数据提取和文本清洗等真实场景中。正则表达式是 AI 工程师进行数据预处理的必备技能，也是后续学习 [文本数据预处理](../../05_数据能力/10_文本数据预处理/) 的重要基础。

## 学习建议

- **学习顺序**：强烈建议按照 01 → 04 的顺序依次学习。模式语法是一切的基础；匹配搜索与替换教你如何在 Python 中实际使用这些模式；分组与断言是进阶语法，能让你的模式更精确更强大；实战文本处理则将前三课所学综合运用到真实场景中。
- **时间分配**：整个主题预计需要 3.5–4 小时。建议第一天完成模式语法和匹配搜索与替换（约 1.5 小时），第二天学习分组与断言（约 50 分钟），第三天完成实战文本处理（约 1 小时）。
- **学习方法**：正则表达式的学习离不开大量动手练习。建议在 Python 交互式环境中边学边试，每学一个新语法元素就立刻用 `re.search()` 或 `re.findall()` 测试效果。推荐使用在线正则可视化工具（如 regex101.com）来辅助理解模式的匹配过程。
- **常见困难点**：
  - 模式语法的元字符较多，初学时容易混淆——不必一次记住所有语法，先掌握最常用的字符类（`\d`、`\w`、`\s`）和量词（`*`、`+`、`?`），其余需要时再查阅。
  - **贪婪匹配与非贪婪匹配**是最常见的困惑来源，尤其在处理 HTML 标签等嵌套结构时。
  - **分组与断言**的语法格式容易记混，建议制作速查卡片辅助记忆。
  - 正则表达式的**回溯机制**可能导致性能问题，在实战中需要注意避免灾难性回溯。
