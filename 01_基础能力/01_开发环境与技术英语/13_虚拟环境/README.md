> **包含知识点**：4 个

---

## 概述

在人工智能和机器学习项目的开发过程中，依赖管理是每位开发者都无法回避的挑战。一个典型的 AI 开发者可能同时维护着多个项目：一个使用 PyTorch 1.13 的经典计算机视觉项目，一个需要 PyTorch 2.x 和最新 Transformers 库的大语言模型实验，还有一个基于 TensorFlow 的老项目需要偶尔修复 Bug。这些项目对 Python 版本、深度学习框架版本、CUDA 工具包版本的要求各不相同，如果全部安装在同一个系统环境中，版本冲突将使你的开发环境迅速变成一团乱麻——安装一个库可能悄悄升级了另一个库的依赖，导致原本正常运行的项目突然崩溃。

**虚拟环境（Virtual Environment）** 正是解决这一问题的核心工具。它允许你为每个项目创建一个独立的、隔离的 Python 运行环境，每个环境拥有自己的 Python 解释器和包集合，互不干扰。本主题将从环境隔离的底层原理讲起，逐步覆盖 `venv` 和 `conda` 两大主流工具的使用方法、依赖固定的最佳实践，最后引入 Docker 容器技术实现更彻底的系统级隔离。掌握这些技能，不仅能让你的日常开发更加顺畅，更是确保 AI 实验可复现性（Reproducibility）的基石——当你或他人需要在另一台机器上复现某个模型的训练结果时，一个精确固定的环境配置将为你省去无数调试的烦恼。

## 学习建议

- **学习顺序**：建议严格按照 01 → 04 的顺序学习。01 环境隔离原理帮助你理解"为什么"需要虚拟环境，这是后续所有操作的动机基础；02 创建与激活是日常开发中使用频率最高的技能；03 依赖固定是团队协作和实验复现的关键；04 Docker 基础属于进阶内容，在前三个知识点的基础上自然延伸到系统级隔离。
- **时间分配**：01 和 02 各约 30–40 分钟，03 约 40–50 分钟，04 约 60–80 分钟（含动手实践），整个主题约需 3–4 小时。
- **学习方法**：虚拟环境是"用中学"的典型主题——建议在学习每个知识点时，都在自己的电脑上实际操作一遍。特别是 venv 和 conda 的创建、激活、依赖导出，只有亲手做过才能形成肌肉记忆。
- **常见困难点**：
  - 初学者容易混淆 `venv` 和 `conda` 的适用场景——前者轻量但只管理 Python 包，后者更重但能管理非 Python 依赖（如 CUDA 工具包）。
  - `requirements.txt` 中版本号的精确固定（pinning）与灵活约束（constraining）的区别，是依赖管理中最容易踩坑的地方。
  - Docker 的概念体系（镜像、容器、Dockerfile）初次接触时会觉得抽象，但一旦动手构建第一个镜像，就会豁然开朗。
