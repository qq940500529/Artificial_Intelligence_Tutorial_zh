## 概述

在前面的课程中，我们学会了用 Python 编写各种程序——从数据处理到网络通信。但你可能已经注意到，有些看似相同的代码运行速度差别很大，有些程序在处理大量数据时会占用惊人的内存。要解答这些"为什么"，就需要深入了解 Python 的 **内存模型（Memory Model）** 和 **性能特征（Performance Characteristics）** 。

本主题不是教你"如何用 Python 写代码"，而是教你"Python 在背后是如何工作的"。理解这些底层机制后，你将能够：

- 写出更高效的代码，避免不必要的内存浪费
- 在遇到性能瓶颈时，知道如何定位和优化
- 理解为什么某些 Python 习惯用法比其他方式更快
- 为后续学习 NumPy、PyTorch 等计算密集型库打下基础

这些知识在处理大规模数据集、训练深度学习模型、以及部署 AI 服务时尤其重要。

## 学习建议

- **学习顺序**：建议按照 01 → 04 的顺序学习。01 对象模型是理解后续所有内容的基础；02 垃圾回收解释了 Python 如何自动释放不再使用的内存；03 性能分析工具帮助你找到代码的"慢点"和"内存大户"；04 内存优化策略是实战技巧的总结。
- **时间分配**：每个知识点预计 40–60 分钟（含动手实践），整个主题约需 3–4 小时。
- **学习方法**：本主题适合边学边实验。建议在 Python 交互式环境中跟着示例一步步操作，使用 `id()`、`sys.getsizeof()` 等工具亲眼观察 Python 的内存行为。
- **常见困难点**：
  - "变量是标签，不是盒子"——这是理解 Python 内存模型最关键的思维转变，需要反复实验来建立直觉。
  - 可变对象与不可变对象的区别——赋值、传参和拷贝行为完全不同，这是很多隐蔽 bug 的根源。
  - 引用计数与循环引用——理解这两个概念才能理解 Python 何时释放内存、何时可能泄漏。
