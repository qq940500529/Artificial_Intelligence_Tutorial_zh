---

## 概述

在编程世界中，文本处理可以说是最常见的任务之一——读取配置文件、清洗数据集、生成报告、解析日志、处理用户输入……几乎每个程序都离不开字符串操作。Python 以其强大而直观的字符串处理能力著称，内置了丰富的字符串方法和灵活的格式化语法，让文本操作变得轻松高效。

然而，字符串不仅仅是"一串字符"这么简单。在全球化的今天，程序需要处理来自世界各地的文字——中文、日文、阿拉伯文、emoji 表情……这背后涉及字符编码的核心概念。理解 Unicode 和编解码机制，是避免"乱码"噩梦的关键。此外，当你需要处理图片、网络数据包或二进制文件时，还需要掌握字节串（bytes）的使用方法。

本主题从实用的字符串方法和格式化技巧出发，逐步深入到 Unicode 编解码原理、字节串操作，最后探讨字符串在 Python 内部的性能特性和优化机制，帮助你建立完整的文本处理能力。

## 学习建议

- **学习顺序**：建议按照 01 → 04 的顺序学习。01 字符串方法与格式化是日常编程中最常用的技能，建议优先掌握；02 Unicode 与编解码是理解"为什么会乱码"的关键，为 03 字节串处理提供理论基础；04 性能与 intern 机制属于进阶内容，适合在前三课掌握后再深入。
- **时间分配**：每个知识点预计 40–60 分钟（含动手实践），整个主题约需 3–4 小时。
- **学习方法**：字符串操作是"多练多记"的技能——方法名记不住没关系，关键是知道"有这个功能"，需要时能快速查文档。建议在学习过程中，用 Python 交互式环境（REPL 或 Jupyter）逐个尝试每个方法。
- **常见困难点**：
  - 初学者容易混淆 `str`（字符串）和 `bytes`（字节串），关键在于理解"文本 vs 二进制"的本质区别。
  - Unicode 编解码的概念比较抽象，建议结合具体的乱码案例来理解。
  - f-string 格式化虽然简单好用，但格式规范符（如 `:.2f`、`:>10`）需要练习才能熟练。
