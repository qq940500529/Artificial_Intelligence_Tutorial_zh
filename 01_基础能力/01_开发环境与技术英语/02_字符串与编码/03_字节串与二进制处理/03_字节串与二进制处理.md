## 正文讲解

### 1. 为什么需要字节串？

在上一课 [Unicode与编解码](../02_Unicode与编解码/02_Unicode与编解码.md) 中，我们学习了 `str` 和 `bytes` 的区别：`str` 存储人类可读的 Unicode 文本，`bytes` 存储原始的字节序列。当时我们主要关注的是文本编码和解码，但 `bytes` 的用途远不止于此。

想想看，计算机中有大量数据本身就不是文本：

- **图片文件**（PNG、JPEG）：像素数据以二进制格式存储
- **网络数据包**：TCP/IP 协议传输的是原始字节
- **音视频文件**：采样数据以二进制编码
- **数据库文件**：用紧凑的二进制格式存储记录
- **加密数据**：密文是看起来毫无规律的字节序列

在处理这些数据时，你需要直接操作字节，而不是文本。这就是 `bytes` 和 `bytearray` 的核心价值。

### 2. bytes 与 bytearray：不可变与可变

Python 提供了两种字节序列类型，它们的关系就像 `str` 和 `list` ：

| 特性 | `bytes` | `bytearray` |
| ---- | ------- | ------------ |
| 可变性 | 不可变 | 可变 |
| 创建方式 | `b"hello"` 或 `bytes(...)` | `bytearray(...)` |
| 类比 | 像 `str` | 像 `list` |
| 适用场景 | 读取、传递数据 | 需要逐字节修改数据 |

```python
# bytes：不可变字节序列
data = b"Hello"
print(type(data))    # <class 'bytes'>
print(data[0])       # 72  —— 返回整数（该字节的值）
# data[0] = 74       # TypeError! bytes 不可变

# bytearray：可变字节序列
buf = bytearray(b"Hello")
print(type(buf))     # <class 'bytearray'>
buf[0] = 74          # 修改第一个字节为 'J'（ASCII 74）
print(buf)           # bytearray(b'Jello')
```

> 💡 **关键区别**：对 `bytes` 进行索引返回的是一个 **整数**（0-255），而不是一个单字符的 bytes 对象。这和 `str` 索引返回单字符字符串不同。

### 3. 创建字节串的多种方式

根据不同的场景，有多种方式创建字节串：

```python
# 方式 1：字面量（仅限 ASCII 可打印字符）
data1 = b"Hello, World!"
print(data1)  # b'Hello, World!'

# 方式 2：用十六进制转义表示任意字节
data2 = b"\x48\x65\x6c\x6c\x6f"   # 等价于 b"Hello"
print(data2)  # b'Hello'

# 方式 3：从十六进制字符串创建
data3 = bytes.fromhex("48 65 6c 6c 6f")
print(data3)  # b'Hello'

# 方式 4：从整数列表创建
data4 = bytes([72, 101, 108, 108, 111])
print(data4)  # b'Hello'

# 方式 5：创建指定长度的零字节
data5 = bytes(10)
print(data5)  # b'\x00\x00\x00\x00\x00\x00\x00\x00\x00\x00'

# 方式 6：从字符串编码获得（上一课学过）
data6 = "你好".encode("utf-8")
print(data6)  # b'\xe4\xbd\xa0\xe5\xa5\xbd'
```

字节串也可以转换回十六进制字符串，这在调试和网络协议分析中非常实用：

```python
data = b"\x48\x65\x6c\x6c\x6f"
print(data.hex())           # '48656c6c6f'
print(data.hex(" "))        # '48 65 6c 6c 6f' —— 用空格分隔
print(data.hex(":", 2))     # '4865:6c6c:6f'   —— 每 2 字节用冒号分隔
```

### 4. 字节串的常用操作

`bytes` 支持很多与 `str` 类似的操作——毕竟它们都是序列类型：

```python
data = b"Hello, World! Hello, Python!"

# 切片
print(data[:5])             # b'Hello'
print(data[-7:])            # b'Python!'

# 查找
print(data.find(b"World"))  # 7
print(b"Hello" in data)     # True

# 替换
new_data = data.replace(b"Hello", b"Hi")
print(new_data)             # b'Hi, World! Hi, Python!'

# 拆分与合并
parts = b"a:b:c".split(b":")
print(parts)                # [b'a', b'b', b'c']
joined = b"-".join(parts)
print(joined)               # b'a-b-c'

# 计数
print(data.count(b"Hello")) # 2

# 判断
print(b"123".isdigit())     # True
print(data.startswith(b"Hello"))  # True
```

对于 `bytearray` ，还额外支持原地修改的操作：

```python
buf = bytearray(b"Hello")

# 追加和扩展
buf.append(33)           # 追加一个字节（'!' 的 ASCII 值）
print(buf)               # bytearray(b'Hello!')

buf.extend(b" World")    # 扩展多个字节
print(buf)               # bytearray(b'Hello! World')

# 插入和删除
buf.insert(5, 44)        # 在索引 5 处插入 ','（ASCII 44）
print(buf)               # bytearray(b'Hello,! World')

# 反转
buf2 = bytearray(b"abc")
buf2.reverse()
print(buf2)              # bytearray(b'cba')
```

### 5. struct 模块：二进制数据的打包与解包

当你需要处理结构化的二进制数据（如文件格式、网络协议）时，`struct` 模块是最强大的工具。它可以将 Python 的基本数据类型（整数、浮点数等）与二进制字节序列互相转换。

#### 基本用法

```python
import struct

# 打包：Python 值 → 字节串
# 'i' = 32位整数，'f' = 32位浮点数，'d' = 64位浮点数
packed = struct.pack("i f d", 42, 3.14, 2.718281828)
print(packed)           # b'...'（二进制数据）
print(len(packed))      # 16 字节（4 + 4 + 8）

# 解包：字节串 → Python 值
values = struct.unpack("i f d", packed)
print(values)           # (42, 3.140000104904175, 2.718281828)
```

#### 常用格式字符

| 格式字符 | C 类型 | Python 类型 | 字节数 |
| -------- | ------ | ----------- | ------ |
| `b` / `B` | signed/unsigned char | int | 1 |
| `h` / `H` | short / unsigned short | int | 2 |
| `i` / `I` | int / unsigned int | int | 4 |
| `q` / `Q` | long long / unsigned | int | 8 |
| `f` | float | float | 4 |
| `d` | double | float | 8 |
| `s` | char[] | bytes | n |
| `?` | _Bool | bool | 1 |

#### 字节序（大端与小端）

不同的计算机架构可能以不同的顺序存储多字节整数——这就是 **字节序（Byte Order）** 问题：

```python
import struct

number = 0x01020304

# 小端序（Little-endian）：低位字节在前（x86 架构常用）
le = struct.pack("<I", number)
print(le.hex(" "))    # '04 03 02 01'

# 大端序（Big-endian）：高位字节在前（网络协议标准）
be = struct.pack(">I", number)
print(be.hex(" "))    # '01 02 03 04'

# 本机字节序
native = struct.pack("=I", number)
print(native.hex(" "))  # 取决于运行平台
```

> 💡 **记忆窍门**：`<` 像箭头指向左边（小端），`>` 像箭头指向右边（大端）。网络协议通常使用大端序（也叫"网络字节序"），所以在处理网络数据时记得用 `>` 前缀。

### 6. 实战：文件魔数检测

每种文件格式通常在文件开头有一段固定的字节序列，称为 **魔数（Magic Number）** 或文件签名，用于标识文件类型。我们可以用字节操作来检测文件类型：

```python
# 常见文件的魔数
FILE_SIGNATURES = {
    b"\x89PNG\r\n\x1a\n": "PNG 图片",
    b"\xff\xd8\xff": "JPEG 图片",
    b"GIF87a": "GIF 图片（87a）",
    b"GIF89a": "GIF 图片（89a）",
    b"PK\x03\x04": "ZIP 压缩包",
    b"%PDF": "PDF 文档",
}


def detect_file_type(filepath):
    """通过文件头的魔数检测文件类型"""
    with open(filepath, "rb") as f:
        header = f.read(8)  # 读取前 8 个字节

    for signature, file_type in FILE_SIGNATURES.items():
        if header.startswith(signature):
            return file_type

    return f"未知类型（文件头：{header.hex(' ')}）"
```

注意 `open()` 中使用了 `"rb"` 模式——这是读取二进制文件的关键。`"r"` 模式会把文件内容当作文本来解码，而 `"rb"` 模式则直接返回原始字节。

## 动手实践

下面这个完整示例演示了如何用 `struct` 模块读写一个自定义的二进制数据文件：

```python
# 文件：code/binary_demo.py
# 使用 struct 模块进行二进制数据的打包与解包

import struct
import os

# 定义数据格式：每条记录包含 ID(int), 分数(float), 姓名(10字节字符串)
# 格式字符串：> = 大端序, I = unsigned int, f = float, 10s = 10字节字符串
RECORD_FORMAT = ">I f 10s"
RECORD_SIZE = struct.calcsize(RECORD_FORMAT)

print(f"每条记录大小：{RECORD_SIZE} 字节\n")

# 准备数据
students = [
    (1, 95.5, "Alice"),
    (2, 88.0, "Bob"),
    (3, 72.3, "Charlie"),
]

# 写入二进制文件
output_path = "/tmp/students.bin"
with open(output_path, "wb") as f:
    for sid, score, name in students:
        # 姓名需要编码为字节并填充到固定长度
        name_bytes = name.encode("utf-8").ljust(10, b"\x00")
        packed = struct.pack(RECORD_FORMAT, sid, score, name_bytes)
        f.write(packed)
        print(f"写入：ID={sid}, 分数={score}, 姓名={name}")
        print(f"  字节内容：{packed.hex(' ')}")

print(f"\n文件大小：{os.path.getsize(output_path)} 字节")
print(f"记录数量：{os.path.getsize(output_path) // RECORD_SIZE}")

# 读取二进制文件
print("\n--- 读取二进制文件 ---")
with open(output_path, "rb") as f:
    while True:
        chunk = f.read(RECORD_SIZE)
        if len(chunk) < RECORD_SIZE:
            break
        sid, score, name_bytes = struct.unpack(RECORD_FORMAT, chunk)
        # 去除姓名的填充零字节并解码
        name = name_bytes.rstrip(b"\x00").decode("utf-8")
        print(f"读取：ID={sid}, 分数={score:.1f}, 姓名={name}")

# 清理临时文件
os.remove(output_path)
```

**运行说明**：
- 环境要求：Python 3.10+
- 运行命令：`python code/binary_demo.py`

**预期输出**：
```
每条记录大小：18 字节

写入：ID=1, 分数=95.5, 姓名=Alice
  字节内容：00 00 00 01 42 bf 00 00 41 6c 69 63 65 00 00 00 00 00
写入：ID=2, 分数=88.0, 姓名=Bob
  字节内容：00 00 00 02 42 b0 00 00 42 6f 62 00 00 00 00 00 00 00
写入：ID=3, 分数=72.3, 姓名=Charlie
  字节内容：00 00 00 03 42 90 99 9a 43 68 61 72 6c 69 65 00 00 00

文件大小：54 字节
记录数量：3

--- 读取二进制文件 ---
读取：ID=1, 分数=95.5, 姓名=Alice
读取：ID=2, 分数=88.0, 姓名=Bob
读取：ID=3, 分数=72.3, 姓名=Charlie
```

这个例子展示了二进制文件的核心工作模式：定义格式 → 打包写入 → 读取解包。在后续学习中，你会发现很多工具（如 NumPy、PyTorch）在底层都使用类似的机制来高效存储数据。

## 典型误区

| 误区 | 正确理解 |
| ---- | -------- |
| `bytes` 和 `str` 可以直接拼接 | 不行！`b"hi" + " world"` 会报 `TypeError` ，必须统一类型 |
| `bytes` 索引返回单字节的 `bytes` | 索引返回的是 **整数**（0-255），切片才返回 `bytes` |
| 二进制文件可以用 `"r"` 模式打开 | 必须用 `"rb"` 模式，否则 Python 会尝试将内容解码为文本 |
| `struct.pack` 的结果是固定的 | 不同字节序前缀（`<`/`>`/`=`）会产生不同的结果 |

## 练习题

### 练习 1：十六进制查看器（难度：⭐）

编写一个函数 `hex_dump(data, width=16)` ，将 `bytes` 对象以十六进制格式打印，类似于 `xxd` 命令的输出。每行显示 `width` 个字节的十六进制值和对应的 ASCII 字符（不可打印字符显示为 `.`）。

例如 `hex_dump(b"Hello, World!\x00\x01\x02")` 输出：

```
00000000  48 65 6c 6c 6f 2c 20 57  6f 72 6c 64 21 00 01 02  |Hello, World!...|
```

<details>
<summary>💡 提示</summary>

使用切片每次取 `width` 个字节，用 `f"{byte:02x}"` 格式化十六进制，用 `chr(byte) if 32 <= byte < 127 else "."` 生成 ASCII 显示。

</details>

<details>
<summary>✅ 参考答案</summary>

```python
def hex_dump(data, width=16):
    for offset in range(0, len(data), width):
        chunk = data[offset:offset + width]
        hex_part = " ".join(f"{b:02x}" for b in chunk)
        ascii_part = "".join(chr(b) if 32 <= b < 127 else "." for b in chunk)
        # 补齐不满一行的情况
        hex_part = hex_part.ljust(width * 3 - 1)
        print(f"{offset:08x}  {hex_part}  |{ascii_part}|")

# 测试
hex_dump(b"Hello, World!\x00\x01\x02")
hex_dump(b"Python 3.10+ \xe4\xbd\xa0\xe5\xa5\xbd")
```

</details>

### 练习 2：简易 BMP 文件头解析（难度：⭐⭐）

BMP 图片文件的前 14 字节是文件头，格式如下：

| 偏移 | 字节数 | 内容 |
| ---- | ------ | ---- |
| 0 | 2 | 魔数 `"BM"` |
| 2 | 4 | 文件大小（字节） |
| 6 | 2 | 保留字段 1 |
| 8 | 2 | 保留字段 2 |
| 10 | 4 | 像素数据偏移 |

编写一个函数 `parse_bmp_header(data)` ，接收至少 14 字节的 `bytes` 对象，解析并返回一个字典：`{"magic": "BM", "file_size": ..., "pixel_offset": ...}` 。

<details>
<summary>💡 提示</summary>

BMP 文件使用小端序（`<`）。格式字符串为 `"<2sIHHI"` ，其中 `2s` 是 2 字节字符串，`I` 是 unsigned int，`H` 是 unsigned short。

</details>

<details>
<summary>✅ 参考答案</summary>

```python
import struct

def parse_bmp_header(data):
    if len(data) < 14:
        raise ValueError("数据不足 14 字节")
    magic, file_size, _, _, pixel_offset = struct.unpack("<2sIHHI", data[:14])
    if magic != b"BM":
        raise ValueError(f"不是 BMP 文件（魔数：{magic}）")
    return {
        "magic": magic.decode("ascii"),
        "file_size": file_size,
        "pixel_offset": pixel_offset,
    }

# 测试（构造一个模拟的 BMP 头）
fake_header = struct.pack("<2sIHHI", b"BM", 1024, 0, 0, 54)
result = parse_bmp_header(fake_header)
print(result)  # {'magic': 'BM', 'file_size': 1024, 'pixel_offset': 54}
```

</details>
