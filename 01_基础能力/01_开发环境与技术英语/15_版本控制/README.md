## 概述

**版本控制（Version Control）** 是软件开发中最基础也最关键的工程实践之一。它帮助开发者记录文件的每一次变更历史，在出错时安全回退，在团队协作中避免互相覆盖代码。无论是个人的小型脚本项目，还是数百人协作的大型 AI 系统，版本控制都是不可或缺的基础设施。

**Git** 是当今最主流的分布式版本控制系统，由 Linux 之父 Linus Torvalds 于 2005 年创建。Git 的分布式架构意味着每个开发者都拥有完整的仓库副本，可以在本地离线工作，完成后再与远程仓库同步。在 AI 项目中，版本控制不仅用于管理代码，还常与实验管理、数据版本控制等工具配合，确保实验结果可复现。GitHub、GitLab 等平台则在 Git 基础上提供了代码审查、持续集成、项目管理等协作功能，是现代软件开发和开源社区的核心基础设施。

## 学习建议

建议按照 01 → 02 → 03 → 04 → 05 的顺序学习，整体约需 3–4 小时。

- **01 仓库与提交** 是所有后续内容的基础，务必亲自动手操作 `git init`、`git add`、`git commit` 等命令，在终端中建立真实的操作记忆。
- **02 分支与合并** 是 Git 最核心的能力，理解分支的本质（指向提交的可移动指针）是掌握 Git 的关键突破点。
- **03 冲突与回滚** 是实际开发中最常遇到的"紧急情况"，建议在学习时刻意制造冲突来练习解决流程。
- **04 标签与版本管理** 相对独立，掌握后可以在自己的项目中立即应用。
- **05 协作工作流** 涉及远程仓库和团队协作，建议在 GitHub 上创建一个练习仓库来实践 Fork + Pull Request 流程。

常见困难点：分支合并时的冲突解决、`git reset` 三种模式的区别、rebase 与 merge 的选择。遇到困惑时，画出提交历史图（或使用 `git log --graph`）会非常有帮助。
