# 并发编程

> **所属路径**：`01_基础能力/01_开发环境与技术英语/07_并发编程`
> **包含知识点**：5 个

---

## 概述

在实际的人工智能工程中，我们经常需要同时处理多项任务——从网络上批量下载数据集、同时预处理成千上万张图片、在模型推理的同时接收新的请求……如果所有任务都一个接一个地排队执行，程序的效率会非常低。**并发编程**就是让程序"同时做多件事"的技术，它是从编程初学者迈向工程实践者的关键一步。

本主题围绕 Python 的三大并发模型展开：**多线程**适合 IO 密集型任务但受 GIL 限制、**多进程**能真正利用多核 CPU 进行并行计算、**异步编程（asyncio）** 以协程方式高效处理海量 IO 操作。在理解这三种模型的原理与适用场景后，我们还将学习 `concurrent.futures` 这一统一的高层接口，最后通过系统对比帮助你在面对不同任务时做出正确的并发模式选择。掌握并发编程不仅能让你的 Python 程序运行得更快，也为后续学习数据加载加速、模型服务部署、分布式训练等人工智能工程核心技能打下坚实基础。

## 学习建议

**推荐学习顺序**：按照 01 → 02 → 03 → 04 → 05 的顺序学习。前三个知识点分别介绍三种核心并发模型，第四个知识点提供统一的高层抽象接口，最后一个知识点进行横向对比与决策指导。建议不要跳过任何一个——即使你觉得自己只需要其中一种模型，了解它们各自的优势和局限才能在实际项目中做出正确选择。

**预计学习时间**：共约 6–8 小时。其中多线程与 GIL 约 1 小时，多进程与进程池约 1.5 小时，异步编程与 asyncio 约 2 小时（这是本主题的重难点），concurrent.futures 约 1 小时，并发模式选择约 1 小时。

**常见困难点**：

- **GIL 的理解**：很多初学者误以为 Python 的多线程完全没用，但实际上它对 IO 密集型任务仍然有效。关键是理解 GIL 只限制了 CPU 密集型任务的并行执行。
- **async/await 的心智模型**：异步编程的思维方式与传统的同步编程截然不同，需要理解"协作式让出控制权"的概念。建议多画执行流程图，用小例子反复练习。
- **选择困难**：面对一个具体任务时，到底该用线程、进程还是协程？第 05 个知识点会给出清晰的决策框架，但前提是你对前三种模型都有基本理解。
