> **包含知识点**：4 个

## 概述

在实际开发中，几乎所有程序都需要和"时间"打交道——记录事件发生的时刻、计算两个时间点之间的间隔、把时间戳转换成人类可读的格式、在不同时区之间换算……Python 标准库提供了 `datetime`、`zoneinfo`、`logging` 等模块来处理这些需求。本主题将带你系统掌握 Python 中日期时间的表示与运算，以及日志系统的搭建与配置，这两项能力是后续并发编程、网络编程和工程化项目的重要基础。

日期时间处理和日志记录看似是两个独立话题，但它们在工程实践中密不可分：日志的每一条记录都需要精确的时间戳，而时区处理不当会导致日志时间混乱、难以排查问题。因此，我们将这两部分放在同一主题中学习，先从基础的日期时间操作入手，再过渡到时区处理，最后深入 Python 的 `logging` 框架及其最佳实践，帮助你建立起完整的"时间 + 日志"工程能力。

## 学习建议

**推荐学习顺序**：按照序号 01 → 02 → 03 → 04 依次学习。前两个知识点聚焦日期时间，后两个知识点聚焦日志，它们之间有自然的递进关系——你需要先理解时间的表示方式，才能正确配置日志中的时间戳格式。

**时间分配**：
- 01 日期与时间处理（约 40 分钟）：内容相对直观，重点在于熟悉 `datetime` 模块的常用类和方法。
- 02 时区与国际化（约 50 分钟）：时区是许多开发者容易踩坑的地方，建议多花时间动手实验不同时区的转换。
- 03 logging 日志框架（约 50 分钟）：需要理解 Logger、Handler、Formatter 三者的协作关系，建议边读边写代码。
- 04 日志配置与最佳实践（约 50 分钟）：涉及 `dictConfig`、日志轮转、结构化日志等工程实践，难度最高，建议结合真实项目场景理解。

**常见难点提醒**：
- **naive 与 aware 的区别**：Python 中不带时区信息的 `datetime` 对象（naive）和带时区信息的（aware）不能直接比较，这是初学者最常遇到的报错之一。
- **日志层级传播**：`logging` 模块中 Logger 的层级关系和消息传播机制容易混淆，建议画图理解父子 Logger 的消息流向。
- **日志配置方式选择**：Python 提供了代码配置、`fileConfig` 和 `dictConfig` 三种方式，推荐优先学习 `dictConfig`，它是目前最灵活也最推荐的方式。
